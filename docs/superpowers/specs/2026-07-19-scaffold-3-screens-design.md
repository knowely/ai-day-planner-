# AI-планер дня — каркас 3 екранів (дизайн)

**Дата:** 2026-07-19
**Статус:** затверджено на етапі брейнштормінгу, готово до плану реалізації

Цей документ замінює структуру екранів із попереднього `2026-07-19-ai-day-planner-design.md` (2 екрани: «Скинути»/«Мій день») на нову, підтверджену структуру з 3 екранів нижче. Технології, модель помилок і філософія YAGNI з попереднього документа лишаються чинними.

## 1. Суть

Каркас mobile-first todo-застосунку без бекенду й без AI-логіки (Фаза 1). Користувач скидає думки текстом (або голосом, де підтримується), розкладає їх в Inbox → Today вручну.

## 2. Екрани та навігація

Нижня таб-навігація, 3 великі кнопки (іконка + підпис), тач-зона ~52px+, фіксована знизу.

### Capture (`/`)
- Заголовок.
- Textarea на весь екран, плейсхолдер «Що в голові?».
- Кнопка мікрофона (велика, кругла): вмикає розпізнавання мовлення через Web Speech API (`SpeechRecognition` / `webkitSpeechRecognition`), розпізнаний текст додається в textarea. Якщо API недоступне в браузері (типово — Safari/iOS) — показати повідомлення «Диктування не підтримується в цьому браузері, введи текст вручну» замість збою.
- Кнопка «Додати» внизу: бере вміст textarea, ріже по рядках, кожен непорожній (після trim) рядок стає новою задачею зі `status: 'inbox'`. Порожній ввід не додає нічого. Поле очищається після додавання.

### Inbox (`/inbox`)
- Список задач зі `status === 'inbox'`.
- Кожен рядок: текст задачі, кнопка «→ Сьогодні» (переводить `status` у `'today'`), кнопка видалення (×).
- Порожній список → текст-заглушка («Тут з'являться твої задачі»).

### Today (`/today`)
- Чекліст задач зі `status === 'today'`.
- Кожен рядок: чекбокс `done` (закреслює текст при позначенні), кнопка видалення (×).
- Порожній список → текст-заглушка.

## 3. Дані

Спільне джерело правди — масив задач у React Context (`useTasks()`), синк із `localStorage` при кожній зміні, завантаження при старті обгорнуте в `try/catch` (биті/недоступні дані → старт з порожнім масивом).

```ts
interface Task {
  id: string
  text: string
  status: 'inbox' | 'today'
  done: boolean
  createdAt: number
}
```

## 4. Технології

Next.js (App Router), TypeScript, Tailwind CSS. Тільки клієнт, без бекенду.

## 5. Обробка помилок

- `localStorage` недоступний/пошкоджений → старт з порожнім списком.
- Порожні (після trim) рядки не додаються як задачі.
- Web Speech API недоступне → повідомлення-fallback, застосунок не падає.

## 6. Ручний чек-лист перевірки каркаса

1. Ввести кілька рядків на Capture, натиснути «Додати» → кожен рядок з'явився окремою задачею в Inbox.
2. В Inbox натиснути «→ Сьогодні» на задачі → вона зникла з Inbox і з'явилась в Today.
3. В Today позначити задачу зробленою → текст закреслюється.
4. Видалити задачу (× в Inbox або Today) → зникає.
5. Перезавантажити сторінку → всі задачі та їх стани збереглись.
6. На Capture натиснути мікрофон у Chrome/Android → починається розпізнавання; в Safari/iOS → показується fallback-повідомлення, застосунок не ламається.
7. Усе зручно тиснути однією рукою на екрані телефону (нижня навігація й кнопка «Додати» в зоні великого пальця).
